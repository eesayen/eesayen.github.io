基础 :

  1. Nodejs 更像一个软件,有安装目录, 打开它的执行文件(window->exe), 会出现一个命令行窗口 ( CLI -> Command-Line Interface ).
    • Node是JavaScript语言脱离浏览器的运行环境
      • 所谓“运行环境”有两层意思：
        1. 首先，JavaScript语言通过Node在操作系统中运行，在这个意义上，Node有点像JavaScript虚拟机；
        1. 其次，Node提供大量工具库，使得JavaScript语言与操作系统互动（比如读写文件、新建子进程），在这个意义上，Node又是JavaScript的工具库。
    • Node内部采用Google公司的V8引擎，作为JavaScript语言解释器；通过自行开发的libuv库，调用操作系统资源。
    • Node.js 是前端工程化的重要支柱之一
  1. node和nodejs就是同一个东西，只是名字不同而已，而npm只是nodejs的一个模块
  1. NPM (node package manager ) 的作用就是对Node.js依赖的包进行管理 , 也可以理解为用来安装/卸载Node.js需要装的东西
    • 传统的HTML,CSS,JS管理太繁杂，模块化就是其中一项成功实践，而npm就是为模块化产生。
    • npm 是 Node.js 官方提供的包管理工具，他已经成了 Node.js 包的标准发布平台，用于 Node.js 包的发布、传播、依赖控制
  1. Vue可以当成是NPM所管理的一个包来安装, Vue.JS源码中,只含有JS代码, 不需要依赖NodeJS就可以运行
  1. 使用webpack跟后台是否用nodejs无关，因为webpack在执行打包压缩的时候是依赖nodejs的，没有nodejs就不能使用webpack
    • webpack就是把 你通过npm中安装的包 打包成更小的浏览器可读的静态资源
  1. vue-cli是一个vue项目的脚手架，可以快速的构建一个vue的基于npm的模块化项目，vue内部的打包机制其实还是借助webpack；

    • 我们还是在逐渐提供一些更多的独立解决方案，比如说路由，状态管理，构建工具链和 CLI

VUE问题整理:

vue.js中常说的数据动态绑定是什么？

就是vue.js会自动响应数据的变化情况，并且根据用户在代码中预先写好的绑定关系，对所有绑定在一起的数据和视图内容都进行修改。而这种绑定关系，在图上是以input 标签的v-model属性来声明的，因此你在别的地方可能也会看到有人粗略的称vue.js为声明式渲染的模版引擎。

前端中为什么要组件化开发？

a、非组件化开发代码和工作量都非常大

前端中如何进行组件化开发？

在面向对象编程中，我们可以使用面向对象的思想将各种模块打包成类或者把一个大的业务模块拆分成更多更小的几个类。在面向过程编程中，我们也可以把一些大功能拆分成许多函数，然后分配给不同的人来开发。

我在很多地方还看到Vuex和Vue-route，它们又是什么？

Vuex是vue的一个状态管理器。用于集中管理一个单页应用程序中的各种状态。

我还在一些地方看到过Vue-resource和Axios，它们又是什么？

我们在传统的前后端不分离的开发中，后端直接把数据通过模版引擎拼接进了返回的HTML中。而现在做单页应用程序属于前后端分离开发，那么这个单页应用程序中的数据就得通过ajax的方式获取，也要通过ajax的方式提交到后端。
