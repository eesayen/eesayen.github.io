模块化 Module

→已知

  1. 须符合同源策略
  1. 页面内部 script 标签叫 type为module, 实现模块,(内嵌程序是模块)
    • type设为module, script标签,内嵌或者scr引用,默认具有defer属性
    • script标签type为module,并且src引用时, 可添加async来修改默认的defer属性
  1. import from " " 实现外部引入
    • import 必须位于顶层代码
  1. 模块内部与外部无关联, 变量名称不受外部影响
  1. export 将模块内部的方法或者变量暴露到模块外面, 被标记为暴露的才可以在模块外面使用
  1. import 将其他模块暴露的方法和变量引入当前JS代码使用
  1. export 和 import 的变量都需要用 { } 包裹起来,且import的变量需要与目标模块export的标量同名
  1. 其中 as用在{ }中 可以分别用在import或export中重名名比变量
  1. 每个模块中可以使用一个export default, 可以export default任意类型, 如 export default { } ->一个对象,引入模块可以使用任意名称变量,接住该对象在引入模块中,通过该名称的对象就可以使用被引入模块的所暴露的所有在对象中的变量
    • 仅在export中有default的情况下,import才可以不使用{ }
  1. 如果在当前模块引入一个模块的变量在暴露,可直接写为 export from
//import {a}  from 'm.js'
//export{a as b}

//如上等同于
export {a as b} from 'm.js'

  • 即便在一个代码块中,引入了两个模块,这两个模块的作用域也是相互独立的, 并不能因为导入到了一个代码块中,就能相互使用对方的变量
  • 要使用模块中不同的独立作用域, 可以按需 导出和导入

  • 可以用 import * as aaa from '' 来导入全部被暴露的变量
  • 因为默认导出只有一个,所以可以使用任何的变量来接收, 不用花括号, 默认导出有多种写法
    //1

//export default {
		...
	}

export { aaa as default}

  • 导入可以将 默认导出的 和 具名导出的 连在一起写
    • 没有使用花括号,表示接受默认导出的
    import aaa, {named1, named2} from ' ' // aaa 为默认导出的, 花括号里为具名
  • 导出也可将默认和具名导出写在同一行
    export {aaa as default, named1 , named2}

  • 一个模块需要引入多个模块
    1. 建立一个中间模块,负责引入所有模块, 在从该模块将引入的全部暴露 ,需要使用的模块中只需引入该模块
    1. 可能遇到变量名重名情况,可用 * 来解决
    import * as a1 from ''
import * as a2 from ''
export {a1,a2}
  • 动态加载模块
    • 因为import必须写在顶层, 不能写在代码块中, 如 if 语句, 函数...
    • import 函数 , 可以放在代码块中,实现按需加载
    import('') // 返回一个Promise对象

补充 - 技术手册

  • import from 语法,只能汇入模块被公开的名称
  • import时指定的名称, 相当于const声明的名称
  • import和export时静态语句,必须撰写在模块的顶层, 不能在if..else..等语句,或者函数中
    • ES11纳入了import( ) 函数,可以用来动态加载模块, 该函数执行传回Promise,任务成功的达成值是Module的实例
    (async function(){
	
	let moduleName=prompt('加载a或b模块?');
	let mod;
	Switch(moduleName){
		case 'a':
			mod = await import('./module-a.js');
		break;
		case 'b':
			mod = await import('./module-b.js');
		break; 
	}
	
	mod.say('xxx')
})()
  • 要公开名称,必须使用export, 此时公开的名称要使用{ } 包含,即使公开的名称只有一个也是如此, 相对的, import时也是如此
  • export的是变量,而不是但是被参考的值 — 此变量名称稍后被指定了新值,另一个模板汇入后,取得的值也会是新值
  • x.js没有定义export default,就不能用以下这种写法
    import aaa from 'x.js'
  • export 公开的是值
    let x = 10;
export default x
x=20 ;   // 汇入此模块的另一模块得到的值并不会是20
