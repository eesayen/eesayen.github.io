ECMAScript规范要求在Object.prototype.toString()调用后,必须传回'[object class]' 格式的字符串

对于内建标准类型,

Object → [object object]

数组 → [object Array]

函数 → [object Function]

...

在ES6以后定义构造函数或类时, 可以定义 symbol.toStringTag特性,来决定 '[object calss]'格式中 'class'的部分

//原型链构造函数定义时
function Foo(value){
	this.value=value
}

Foo.prototype[symbol.toStringTag]='Foo'

Object.prototype.toString.call(new Foo()); // ->'[object foo]'

//使用类语法时
class ImmutableList{
	constructor(){
	}
	get [symbol.toStringTag](){
		return this.constructor.name;
	}
}

Object.prototype.toString.call(new ImmutableList())// ->'[object ImmutableList]'

事实上, Array.isArray( ),也是根据内部实现特性[[class]]的值来判断

symbol.species协议指定子类要使用哪个类来创建实例

Reflect 和 Proxy

  1. ECMAscript规范了[[GetPrototypeOf]],[[SetPrototypeOf]], [[IsExtensible]],等运算, 在实现JavaScript引擎时,这类运算实现为 引擎的内部方法, [[OwnPropertyKeys]]也是其中之一
  1. ES5时, 可以存取内部方法的API随意的配置为Object 函数,
    1. ES6以后提供了 Reflect API作为存取内部方法的统一管道,并提供更多控制细节
    1. 也新增Proxy API 提供干涉内部方法的时机点, 与可干涉的 Reflect API 一对一对应
    1. Proxy实例可捕捉内部方法被操作的时机, 在处理器中, Reflect用来存取内部方法, 两者结合就成了meta-programming的重量级武器

    ES5时引擎内部的方法API配置为Object函数,ES6后调整并以Reflect作为名称空间

    Refelct.getPrototypeOf();
Reflect.setPrototypeOf();
Reflect.isExtensible();
Reflect.preventExtensions();
Refelct.defineProperty()
Refelct.getOwnPropertyDescriptor()
...

    功能上基本相同, 略微调整在于

    • Obejct有些对应函数执行后传回对象本身, Reflect传回布尔值
    • Object对应函数target参数若是基本类型,会使用对应类实例来包裹, Reflect此时会抛出TypeError

  1. ES6以后, 建议使用Reflect的这些方法来取代Object对应的方法, 未来与内部方法存取相关的函数，只会在Reflect上配置, 例如 Reflect.ownKeys( )
    Reflect.ownKeys(regExp.prototype)// 相当于 Object.getOwnPropertyNames()和getOwnPropertySymbols()串接后的清单

  1. 控制函数中的this对象, 可以使用函数实例的apply( )和call( ) 方法,精确来说,是使用 Function.prototype上定义的函数, ES6以后, 可以使用 Reflect.apply( )
  1. 建立对象时,可以定义取值和设置函数, 既然时函数,其中就可以有this,如何控制取值和设置中的this对象呢.
    1. →ES5时,必须取得特性描述器→从中取得get,set特性,得到函数→运用call( ),apply( )等方法
    1. ES6以后,可以用 Reflect.get( ), Reflect.set( )方法
    let o ={
	x:10,
	get doubleX(){
		return this.x*2;
	}
}
//ES5
let doubleFn=Object.getOwnPropertyDescriptor(0,doubleX).get
doubleFn.apply({x:20});// -> 40

//ES6
Reflect.get(o,doubleX) //->20
Reflect.get(o,doubleX,{x:30}) //->60

    实际上, Reflect.get( ), Reflect.set( )并非只是在用 取值 设值 函数, Reflect API是用来存取内部方法的管道. Reflect.get( ), Reflect.set( )存取的是 [[GET]], [[SET]] 内部方法.

    纯粹用来取值也是可以,例如

    Reflect.get(Array.prototype,'forEach')//-> [Function:forEach]
Reflect.get(Array,'of')//-> [Function:of]

  1. 运算符 in delete new 对应的Reflect函数
    1. in delete new运算符会使用到内部方法, Reflect有这类运算符对应的函数
      1. Reflect.has( )
      1. Reflect.deleteProperty( )
        1. delete删除失败会传回FALSE,严格模式下会引发 TypeError
        1. Reflect只会传回true or false
      1. Reflect.constructor( )
        1. 第一参数:　构造函数　或　类
        1. 第二参数：数组, 给第一参数传递的参数

        Reflect.constructor( )能做的事情远比 new 多, 因为可以指定构造函数中new.target的实际构造函数或类, new.target 代表构造函数 ???

        构造函数的prototype会成为实例的原型

  1. 如想对对象特性存取时进行一些操作, 如console.log( ) ,
    • 可以设置取值,设置函数, 但是有两个问题
    1. 建立对象的控制权可能不在你手上,如链接其他库
    1. 如果不打算进行操作, 需要去代码里面拿掉,也代表了难以重用

    → 更好的解决 :

    ES6可通过new Proxy(target, handle)建立Proxy实例, 操作时, Proxy实例会捕捉并调用handle 相对应方法

    为做具体控制, Proxy API 经常与 Reflect API搭配使用, Reflect 的方法与 Proxy的不足一对一对应

  1. Proxy API来实现代理模式
    1. 实现上, 提供代理对象给客户端操作
    1. 代理对象与目标对象具有相同的接口
    1. 代理对象可以单纯的将任务转发给目标对象,或者附加额外的逻辑
    • 代理模式的原则是:　客户端对实现不知情，但JS提供一定程度的检测,若明显违反既有协议试图实现代理,会抛出TypeError

  1. this指向问题
    1. 目标对象方法中有this, 通过Proxy实现时,this默认指向Proxy实例
    1. 但是用this作为取得另一数据依据时, 要注意,若因this 出现错误, 可以通过bind( )方法将this绑定原来的目标对象

  1. 可撤销的 Proxy实例
    • new Proxy实例 →持有实例→持续代理实例
    • 使用Proxy.revokable( target obj , handle )函数建立可撤销的实例

      →返回一个对象,具有

      • proxy属性 - 保存了通过new Proxy() 建立的实例
      • revoke( )方法 - 调用此方法撤销代理

      • 调用revoke( )后,在操作proxy属性保存的方法会抛出TpyeError
