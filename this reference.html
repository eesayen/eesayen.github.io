• 函数在对象里面的时候,是对象的属性,也叫对象的方法,
  • 当一个方法被调用的时候,此时this绑定到调用该方法的对象上
  • 函数调用模式
  • 以此模式调用函数式,对象绑定到全局(严格模式下会有不同)
  • 箭头函数不会有自己的this,他会从作用域链的上一层继承this
  • 构造器调用模式
  • 函数前带一个 NEW 运算符来调用,该函数是一个构造函数,此时是构造器,
  • this指向生成的空的对象
  • apply , call, bind(--返回新函数) 调用模式
  • this绑定到它们第一个参数指定的对象上
  • this的值不是固定的,根据不同的调用方式,绑定到不同的对象上
  • this的绑定规则,简单来说,函数执行时的上下文是谁,this就指向谁
  • this依调用方式而定
  • 箭头函数的this不是根据调用者来绑定的,而是看当时的词汇环境来绑定的,且一旦绑定就不会更改

this一般用在面向对象的编程中

箭头函数解决了函数调用模式this为undefined的问题

  • this属于运行时的对象，也就是说只有当前函数被调用了，this才会有意义(被赋值)
  • this 在不同的环境中可以指代不同的东西,如果在对象的方法中使用 this，而该方法在该对象的上下文中调用，那么 this 指代该对象本身。
  • 在定义箭头函数时，不管 this 指向谁，箭头函数内部的 this 永远指向同一个东西
